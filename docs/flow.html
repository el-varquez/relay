Workflow

Hand it an approved plan or handoff. It scouts, builds, tests, waits for your review (that's where QA happens), then ships — auto-looping on failure. One project-agnostic tool that reads each project's own CLAUDE.md to learn how that project builds.

Scout → Build → Test → Review → Ship 2 read-only verifiers · 1 writer loop cap: 3 rounds

Two read-only verifiers bracket the one writer: Scout verifies the plan before, Test verifies the result after. Read-only is enforced by tool scope — Scout and Test have no edit access at all.

03

Rules baked in

◆ Project-agnostic

Nothing hardcoded. The orchestrator reads this project's CLAUDE.md for build/test commands and git conventions. No merged multi-project config.

◆ QA is part of Review

Approval = QA + engineer satisfied. That sign-off is what authorizes Ship — nothing merges before a human (and QA) says so.

◆ Persistent Build

The same Build agent is continued across retries, so it remembers prior attempts. Both fail arrows feed it. Cap 3 rounds, then escalate.

◆ No Claude co-author

Built into the workflow — no Co-Authored-By: Claude trailer on commits. Add it to your own ~/.claude/CLAUDE.md to enforce it everywhere.
